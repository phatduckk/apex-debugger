On a Neptune Apex, those raw values in the status JSON/XML for digital inputs (like optical/float switches) are a little unintuitive because they’re not simple booleans—they’re bitmask-style state values coming from the I/O subsystem.
Here’s what you’re seeing:

What the values mean
* 0 → OPEN / not triggered
* 60 → CLOSED / triggered (normal state for many optical sensors)
* 200 → CLOSED / triggered (alternate state or different port/module encoding)


Why there are different “closed” values
Apex doesn’t just send 0 or 1. Instead, it encodes multiple bits of info into that number:
* switch state (open/closed)
* module type (FMM, base unit, etc.)
* port number / input type

So:
60 and 200 both mean “ON/CLOSED”, just coming from different modules or input types
This is why you’ll often see inconsistent “true” values across switches


Practical takeaway (important)
Don’t key off the exact number—just treat it like:
const isClosed = value !== 0;

That’s the safest way to normalize it across:
* FMM optical sensors
* float switches
* leak detectors

Bonus tip (useful for your extension)
If you're building logic for your debugger:
* Normalize all switch inputs to boolean early
* Optionally map:
  * 0 → false
  * >0 → true

Core Truth (don’t skip this)
Before the table:
* 0 = OPEN (OFF)
* ANY NON-ZERO = CLOSED (ON)
Everything else is just metadata baked into the number.



Value	Meaning	Notes
0	OPEN / OFF	Switch not triggered
60	CLOSED / ON	Very common (FMM optical, float switches)
200	CLOSED / ON	Also common (different module/port encoding)

⚙️ What’s Actually Going On (bit-level)
These numbers are combinations of flags like:
* Switch state bit
* Module type (FMM, base, etc.)
* Input port index
* Sensor type (optical vs mechanical vs leak)

Example intuition (not exact docs, but accurate behaviorally):
* 60 → one combo of bits for “closed” on FMM
* 200 → different combo for “closed” on another port/module
* 0 → all bits off → open


FMM (Fluid Monitoring Module)
* Optical sensors → usually 60
* Leak detectors → often 100–200 range
* Some ports → 200
Base Unit / Breakout Box
* Float switches → often:
  * 0 (open)
  * 4, 12, 20, 28 (closed variants)

Normalize like this:
function normalizeSwitch(value) {
  return value !== 0;
}



Important Gotcha
Two switches can both be CLOSED but report:
one → 60
one → 200
That does NOT mean different states—just different hardware paths.
